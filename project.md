# 프로젝트 개요: 인턴 경험 일지 블로그

## 1. 프로젝트 소개

이 프로젝트는 인턴 경험을 **하루 단위로 기록**하는 블로그/일지 사이트다.

동시에 다음 기술 스택을 실전으로 연습하는 것을 주요 목표로 한다:

- React
- Supabase (DB, Auth, RLS)
- Vercel (배포)
- Tailwind CSS
- shadcn/ui

사용자는 이메일 로그인(또는 매직 링크)으로 접속하여, 날짜별 인턴 일지(제목, 본문, 태그 등)를 작성·수정·삭제할 수 있다.

## 2. 기술 스택

- **프론트엔드**: React
- **백엔드/BaaS**: Supabase (PostgreSQL, Auth, Row Level Security)
- **배포**: Vercel
- **스타일링**: Tailwind CSS
- **UI 컴포넌트**: shadcn/ui

## 3. 핵심 기능

- 이메일 로그인 또는 매직 링크 로그인 (Supabase Auth)
- 로그인한 사용자별 **개인 일지** CRUD
- 하루 1개 이상 일지 작성 가능, 태그/카테고리 필터
- 반응형 레이아웃, 다크 모드 지원 (가능하다면)
- 프로덕션 배포(Vercel) 및 커스텀 도메인(옵션)
- **AI Diary Agent (New)**: Vercel AI SDK (Google Gemini) 기반 실시간 일기 작성 협업 및 썸네일 생성

## 4. 학습 목표

- Supabase를 이용해 **인증 + RLS 기반 개인 데이터 저장 구조** 이해하기
- React에서 Supabase 클라이언트로 데이터 fetch / mutate / 실시간 동기화 하는 패턴 익히기
- Tailwind CSS 유틸리티와 shadcn/ui 컴포넌트 구조를 이해하고, 필요한 부분을 커스터마이징하는 연습
- Vercel로 배포 파이프라인(Git 연동, 환경 변수 설정 등)을 실제로 구성해 보기

## 5. antigravity를 위한 Instruction

이 프로젝트는 “빠르게 완성된 결과물”보다 **학습 경험**이 더 중요하다.

따라서 답변/코드를 줄 때 다음 원칙을 꼭 지켜 달라:

1. **설명 + 실습 형태로 답변하기**
    - 단순히 코드만 주지 말고, 먼저 개념/이유를 짧게 설명한 뒤
    “1단계: …, 2단계: …”처럼 **직접 따라 해 볼 수 있는 실습 단계**를 제시해 달라.
    - 예:
        - RLS를 설정할 때, “왜 이 정책이 필요한지 → Supabase 콘솔에서 어떻게 설정하는지 → 예시 SQL → React에서 어떻게 검증할 수 있는지” 순서로 안내.
2. **최소 예제부터 점진적으로 확장**
    - 처음부터 복잡한 구조를 주지 말고,
        - (1) 최소 페이지/컴포넌트
        - (2) 간단한 Supabase 연동
        - (3) 인증 추가
        - (4) RLS/멀티테넌트적 구조
        순서로 점진적으로 확장하는 식으로 가이드해 달라.
3. **현재 실력 수준을 가정**
    - React 기본 개념(컴포넌트, props, hooks)은 알고 있다는 가정.
    - Supabase/Auth/RLS, shadcn/ui, Tailwind는 아직 익숙하지 않다고 가정하고,
    개념 설명을 한두 문단 정도 먼저 써 준 뒤 코드 예제를 제시해 달라.
4. **선택지와 트레이드오프도 함께 설명**
    - 예:
        - “일지 테이블을 `user_id` 컬럼 하나로 멀티테넌트 처리할지, 스키마를 나눌지” 등 설계 선택지와 장단점을 짧게 비교해 달라.
        - Tailwind 유틸리티만 쓸 때와 shadcn/ui를 섞을 때 구조가 어떻게 달라지는지 짧게 언급.
5. **매일 기능을 조금씩 확장하는 식의 TODO 제안**
    - 답변 끝에, “오늘은 여기까지, 다음 날에 해 볼 만한 TODO 2~3개”를 제안해서,
    인턴 일지와 함께 **하루 단위로 개발 연습**을 이어갈 수 있게 해 달라.
6. **코드는 가능한 한 최신 베스트 프랙티스를 기반으로 작성**
    - React 함수형 컴포넌트, hooks 기반.
    - Supabase의 최신 JS 클라이언트 사용.
    - Tailwind/shadcn 최신 구조(예: 앱 디렉토리 구조를 사용하는 경우에는 그에 맞는 파일 예시) 반영.
